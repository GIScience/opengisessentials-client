Es ist möglich sich aktiv in der Karte mittels eines "OpenLayers" Klienten zu bewegen (format=application/openlayers). Natürlich kann auch ein anderes Format gewählt werden (z.B. format=image/png). Eine Übersicht über die unterstützten Formate finden Sie in den Capabilities.

Sie haben die Möglichkeit die Felder "WMS" und "SLD" entsprechend den OGC Spezifikationen zu verändern. Sie können die URL Parameter der WMS Anfrage frei editieren. Ebenso ist es möglich eigene URLs einzugeben. Bitte beachten Sie, dass Sie dann gegebenenfalls auch andere Parameter (Layer, Koordinaten, Referenzsystem, Format, etc.) anpassen müssen.

Interessanter dürfte die Möglichkeit sein das SLD Dokument anzupassen (Bereich SLD), deshalb hier ein kurzes Beispiel.

Sie haben das Beispiel mit der einfachen Liniensymbolisierung aufgerufen. Jetzt möchten Sie testen wie dieses mit einem Strichmuster aussehen würde. Schauen Sie sich im Buch die SVG Styling Parameter an (Seite 214), oder schlagen Sie in der Spezifikation nach.

Tragen Sie den gewünschten Parameter in den SLD Bereich ein, z.B.:

...

<Stroke>
<SvgParameter name="stroke">#0075FF</SvgParameter>
<SvgParameter name="stroke-width">6</SvgParameter>
<SvgParameter name="stroke-opacity">1.0</SvgParameter>
<SvgParameter name="stroke-dasharray">8 10</SvgParameter>
</Stroke>

...

Dann klicken Sie den Button "Karte ausgeben" an und die Karte wird ausgegeben.

Mit den Informationen aus den Tabellen mit den Attributen der Geodaten lassen sich auch komplexe SLD Dokumente mit integrierten Filter Encoding Elementen erstellen.

Bitte beachten Sie das die hier verwendete Kartenserver Software den OGC Standard Styled Layer Descriptor 1.1.0 bisher nicht zu 100 % unterstützt. Es kann daher vorkommen, dass eigene Editierungen aus diesem Grunde nicht funktionieren, obwohl diese dem OGC Standard entsprechen. So konnte z.B. die Flächensymbolisierung mit Graphic Fill Elementen nur mit SLD 1.0.0 realisiert werdfen.

Bitte beachten Sie ebenfalls, dass die Länge des Requests durch die Wahl Ihres Browsers eingeschränkt sein kann. Mit dem Internet Explorer 8 sind beispielsweise nur http-Anfragen von bis zu 2083 Zeichen möglich. Aus diesem Grunde sollten Sie den Firefox Browser benutzen, da mit dieser freien Software wesentlich längere Anfragen gestellt werden können.
